{# Activation Command-Box — reusable shared component (REVUE-384). A dark card with a brand-purple border, a monospace command line, and a Copy button that flips to "Copied! ✓" for ~2s (reusing copyToClipboard from base.html). Consumed by /activate (default state, full command visible) and, later, the Account→Plan page (masked state). The macro emits ONLY the static skeleton + a `data-copy-payload` attribute; the actual command text and copy payload may be set server-side (params) or client-side (shared JS init). States: default — `command` shown verbatim; Copy yields `copy_payload` (or `command`). masked — `masked_display` shown (e.g. lic_••••1234) while Copy still yields the FULL `copy_payload`. The full value never appears as visible text — only inside the `data-copy-payload` attribute. copied — transient label state owned by copyToClipboard, not rendered here. Params: dom_id — id for the card element (defaults to "command-box"; the copy button id is derived as "-copy"). Restrict to HTML-id-safe characters; it is only ever rendered into autoescaped HTML-attribute contexts (never a JS string). command — the visible command line (default state). copy_payload — the exact string Copy writes to the clipboard. Defaults to `command` when omitted. caption — small helper text under the command (optional). masked — when true, render `masked_display` instead of `command`. masked_display— the masked visible text (e.g. lic_••••1234). Caller contract: in masked mode the caller MUST pass `copy_payload` (the full unmasked value). Omitting it makes Copy yield the empty `command` default — the masked display would have nothing to copy. #} {% macro command_box(dom_id="command-box", command="", copy_payload=none, caption="", masked=False, masked_display="") %} {% set _payload = copy_payload if copy_payload is not none else command %} {% set _visible = masked_display if masked else command %}
