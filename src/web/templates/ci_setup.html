CI Setup

This page covers CI mode: automated reviews on PRs/MRs. For local/pre-commit reviews use the CLI: revue activate <key>.

New to the CLI? See the CLI activation guide to get started in your terminal.

CI mode needs four pieces: your Revue licence key, an AI-provider API key, a pipeline YAML step, and provider/model config. Across all three platforms Revue uses one provider-key variable name: AI_API_KEY. The setup is identical regardless of which AI provider you pick.

{{ step_block( 1, "Add your licence key as a repository variable", 'Go to Repository settings → Repository variables and add REVUE_LICENSE_KEY (mark it Secured).' ) }} {{ step_block( 2, "Add your AI-provider API key", 'In the same Repository variables screen add AI_API_KEY (Secured). This one variable name works for every provider.' ) }} {{ yaml_step(3, "Add bitbucket-pipelines.yml", "bitbucket", BITBUCKET_YAML) }} {{ provider_step(4, "bitbucket", "anthropic", "claude-sonnet-4-5") }}
{{ step_block( 1, "Add your licence key as a secret", 'Go to Settings → Secrets and variables → Actions → New repository secret and add REVUE_LICENSE_KEY.' ) }} {{ step_block( 2, "Add your AI-provider API key", 'Add another secret named AI_API_KEY in the same screen. One name, any provider.' ) }} {{ yaml_step(3, "Add .github/workflows/revue.yml", "github", GITHUB_YAML) }} {{ provider_step(4, "github", "anthropic", "claude-sonnet-4-5") }}
{{ step_block( 1, "Add your licence key as a CI/CD variable", 'Go to Settings → CI/CD → Variables → Add variable and add REVUE_LICENSE_KEY (Masked).' ) }} {{ step_block( 2, "Add your AI-provider API key", 'Add another variable named AI_API_KEY (Masked). The same name works whichever provider you use.' ) }} {{ yaml_step(3, "Add .gitlab-ci.yml", "gitlab", GITLAB_YAML) }} {{ provider_step(4, "gitlab", "openai", "gpt-4o-mini") }}
Prefer local reviews? Run revue activate <key> and review before you even open a PR. See the CLI activation guide.
